# 要件定義書

最終更新: 2025-02-08

## 1. システム概要

Coordyは、インストラクター/事業者がサービスを登録し、ユーザーがオンライン・オフラインでの予約・決済を行うプラットフォーム。外部サイトとのAPI連携によるゲスト予約にも対応する。

### 技術スタック

| 項目 | 技術 |
|------|------|
| フレームワーク | Next.js 16 (App Router) |
| ランタイム | React 19 |
| 認証 | Supabase Auth (JWT) |
| DB | PostgreSQL (Supabase) |
| ORM | Prisma 6 |
| 決済 | Stripe API |
| メール | Resend |
| ビデオ通話 | Google Meet (Calendar API) |
| スタイリング | Tailwind CSS |
| 言語 | TypeScript |

---

## 2. ロールと権限

| ロール | 説明 | 主な操作 |
|--------|------|---------|
| USER | 一般ユーザー | サービス閲覧・予約・決済・プロフィール管理 |
| INSTRUCTOR | インストラクター/事業者 | サービス作成・スケジュール管理・予約管理・売上管理 |
| ADMIN | 管理者 | ユーザー管理・本人確認審査・チャージ承認・パートナー管理 |

- 同一メールアドレスで複数ロール登録可能（email+role ユニーク制約）

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| 認証セキュリティ | Supabase JWT + RLS、HMAC署名検証（外部API） |
| データ保護 | 銀行口座番号の暗号化（AES-256） |
| 決済安全性 | Stripe PCI DSS準拠、3Dセキュア対応 |
| タイムゾーン | Asia/Tokyo デフォルト |
| 言語 | 日本語（将来多言語対応予定） |
